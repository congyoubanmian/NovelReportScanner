## 剩余事项

### 1. 不建议机械照搬的字段命名差异

Kimi 草案里仍有少量字段名与当前项目不一致，例如：

- `apocalypse_survival`
  - Kimi：`humanity_and_morality`、`power_system`、`exploration_and_adventure`
  - 当前：`humanity_moral_dilemmas`、`power_evolution_system`、`exploration_adventure`
- `crime_forensics`
  - Kimi：`social_relevance`
  - 当前：`social_reflection`
- `isekai_lightnovel`
  - Kimi：`adventure_structure`、`companions`
  - 当前：`adventure_system`、`party_dynamics`

这些多半是命名风格差异，当前字段更贴合已有报告标题和测试，不建议为了完全一致而改名，除非同步迁移报告字段映射和历史结果兼容。

### 2. 剩余 scan_focus 文案差异

仍有少量 Kimi 原文与当前文案不完全一致，例如：

- 末世：部分“常见风险”“末世经济体系”表述可继续精炼。
- 刑侦：可继续补“程序违法无后果、法医技术神化”等风险提示。
- 克系/推理/体育：部分句子已经语义覆盖，但和 Kimi 原句不完全相同。

这些属于低风险提示词打磨，不影响当前功能主线。

### 3. 工程优化报告中的后端项

`OPTIMIZATION_REPORT.md` 里的工程项尚未全部完成，建议后续单独拆批处理：

- 上传文件大小限制与分块写入。
- `/api/state` 锁内 IO 和 CPU 工作过重。
- `_find_book_outputs` 遍历 results 的性能优化。
- `_save_state()` 原子写入。
- SSE 替代轮询。
- 请求超时、CORS、生产 HTTP 服务稳定性。
- Docker 非 root 用户运行。

这些属于工程质量/性能安全方向，和 Kimi profile 配置优化不同，建议单独开批次做，避免和扫描规则混在一起。

### 4. 真实书籍回归

后续最有价值的是用真实书籍报告继续校准：

- 后宫：女主重复、漏女、送女/绿帽误判。
- 多标签：历史+科幻+后宫、蒸汽西幻+探案、异世界+经营等混合类型。
- 非后宫类型：确认报告输出是否真正体现专项字段，而不是只在 prompt 里出现。
