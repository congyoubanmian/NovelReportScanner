# 扫书优化 V2 路线图 — 感性量化 × 可信度 × 模拟报告

> 生成时间：2026-06-10
> 核心理念：写作与阅读虽是感性体验，但可以通过「多维信号采样 → 统计聚合 → 置信度评估 → 模拟读者报告」的方式量化呈现
> 目标：让扫书报告不仅是信息摘要，更是「模拟一位老练读者给你写书评」

---

## 设计哲学

### 为什么能量化？

感性体验看似主观，但存在可观测的客观信号：

1. **文字信号** — 模板词频率、句式重复率、信息密度 → 反映"词穷/水文"
2. **结构信号** — 章节长度方差、场景密度、伏笔回收率 → 反映"节奏/结构"
3. **情绪信号** — 爽/虐/燃/悲/悬分布、高潮间隔 → 反映"情绪调动"
4. **人物信号** — 角色出场频次衰减、关系网密度 → 反映"人物塑造深度"
5. **阅读投入度** — 连续低投入区域占比 → 反映"可读性断崖"

### 量化 ≠ 纯算法

我们不走纯算法路线（工程量巨大且不可靠），而是 **LLM判断 + 规则统计 双通道融合**：

- **LLM 判断通道**：让模型在 chunk 级别输出结构化评分（已有 radar_scores、pacing_analysis 等）
- **规则统计通道**：程序从原文直接计算文本特征（模板词频率、信息密度等）
- **融合层**：两路信号交叉验证，LLM 评分与程序统计偏离过大时降权，产生置信度

### 可信度分级

每个量化指标都附可信度标签：
- **🟢 高可信**（≥80%）— LLM 评分与程序统计一致
- **🟡 中可信**（50-80%）— 只有一路信号支持
- **🔴 低可信**（<50%）— 信号不足或互相矛盾

## 实现计划

### P0: 文学质感量化引擎 `literary_metrics.py`

**纯规则统计，零 LLM 成本，从原文直接计算**

#### 文本质感指标
| 指标 | 计算方式 | 含义 |
|------|----------|------|
| 模板词密度 | 已有 `STAGE1_WORD_POVERTY_MARKERS` 词频 / 总字数 × 10000 | 越高越"套路" |
| 信息熵/千字 | 去重字符集大小 / 总字符数 | 越低越"水" |
| 对话占比 | 引号内字符 / 总字符 | 反映叙事方式 |
| 场景切换密度 | 章节内时间/地点跳跃标记 / 章 | 反映节奏 |
| 长句率 | 超过 N 字的句子占比 | 文笔特征 |
| 感叹号密度 | ！/总字数 × 1000 | 情绪外露程度 |
| 省略号密度 | ……/总字数 × 1000 | 留白风格 |

#### 结构性指标
| 指标 | 计算方式 | 含义 |
|------|----------|------|
| 章节长度变异系数 | std/mean × 100 | 越大越不均匀 |
| 长章占比 | >均值×1.5 的章占比 | 节奏风险 |
| 短章占比 | <均值×0.5 的章占比 | 可能是水文 |
| POV 切换频率 | 视角标记次数 / 章 | 叙事复杂度 |

### P1: 速读卡片

**在报告最前面插入一张"速读卡片"，像豆瓣读书评分 + 知乎书评摘要**

```
┌─────────────────────────────────────────┐
│ 📖 速读卡片                              │
│                                          │
│ 综合推荐：A 级（强推）                    │
│ 置信度：🟢 高（基于 120 个采样片段）       │
│                                          │
│ ◆  一句话评价：XX特色鲜明，XX处理出彩，    │
│    但XX略显不足                           │
│ ◆  类型定位：玄幻·升级·热血               │
│ ◆  适合人群：喜欢升级爽文、不太纠结感情线  │
│ ◆  避雷提示：XX情节较重、后期节奏偏慢     │
│                                          │
│ 文本质感：■■■■□□□□□□ 4.2/10             │
│ 信息密度：■■■■■■□□□□ 6.1/10             │
│ 节奏把控：■■■■■■■□□□ 7.3/10             │
│ 情绪调动：■■■■■■■■□□ 8.1/10             │
│ 人物塑造：■■■■■□□□□□ 5.5/10             │
│ 世界观：  ■■■■■■■□□□ 7.0/10             │
│                                          │
│ 🔥 高潮章节：第12章、第38章、第67章       │
│ 😐 低谷区域：第45-52章（日常过渡段）       │
│ ⚠️ 风险提示：后期战力膨胀、女配工具人化   │
└─────────────────────────────────────────┘
```

### P2: 量化评分融合

**将 LLM radar_scores 与 literary_metrics 纯规则统计交叉验证**

- LLM 给了 `writing: 8.0`，但程序发现模板词密度 15（很高） → 写作评分降权，标注「🟡 中可信」
- LLM 给了 `pacing: 6.5`，程序算出章节长度变异系数 35% → 一致，标注「🟢 高可信」
- LLM 给了 `emotion: 9.0`，但程序发现爽/虐分布均匀 → 情绪评分降权

### P3: 模拟读者书评段

**在报告末尾生成一段模拟读者书评，增加可读性**

不是冰冷的指标列表，而是模拟一个"读完此书的老读者"的口吻：

```
【模拟读者书评】
"这本书前 200 章是真的爽，升级节奏紧凑，每次觉得要卡住了就来个反转。
但从第 250 章开始节奏明显放慢，日常戏偏多，连续 30 章没有大的剧情推进。
世界观设定挺有意思，尤其是 XX 体系的设计很有想法，但后期战力体系有点崩。
总的来说，如果你喜欢升级爽文，前半本绝对值得看，后半本可以跳着看。"
```

### P4: 置信度报告段

```
【量化可信度报告】
  文本质感评分：4.2/10  🟢 高可信
    → LLM 评分 4.0 与模板词统计密度 15.2/万字 一致
  信息密度评分：6.1/10  🟡 中可信
    → 仅有 LLM 评分，程序采样片段不足
  节奏把控评分：7.3/10  🟢 高可信
    → LLM 评分 7.5 与章节长度变异系数 28% 一致
  ...
```

## 实现细节

### 新增文件

| 文件 | 功能 |
|------|------|
| `literary_metrics.py` | 纯规则文本质感量化：模板词密度、信息熵、对话占比等 |

### 修改文件

| 文件 | 改动 |
|------|------|
| `report.py` | 新增速读卡片段、量化融合段、模拟书评段、可信度报告段 |
| `general_scan.py` | chunk 级别增加 literary_metrics 采样 |

### 配置变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `LITERARY_METRICS_ENABLED` | `1` | 文学质感量化 |
| `SPEED_READ_CARD_ENABLED` | `1` | 速读卡片 |
| `SIMULATED_REVIEW_ENABLED` | `1` | 模拟读者书评 |
| `CONFIDENCE_REPORT_ENABLED` | `1` | 置信度报告 |
| `LITERARY_METRICS_SAMPLE_CHARS` | `300000` | 采样字符数 |

## 回退方案

所有新功能均有配置开关，可独立关闭：

```bash
LITERARY_METRICS_ENABLED=0
SPEED_READ_CARD_ENABLED=0
SIMULATED_REVIEW_ENABLED=0
CONFIDENCE_REPORT_ENABLED=0
```
