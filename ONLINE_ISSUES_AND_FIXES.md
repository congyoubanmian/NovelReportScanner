## 一、发现的线上问题

### 1.1 7本后宫书标记completed但无报告文件 ⚠️

**影响书籍**：九锡、这个剑修有点稳、庶子无敌-上汤豆苗、混江湖的谁谈恋爱啊、晋末长剑、仙子请听我解释、鸣龙

**症状**：`status=completed` 但 `output_index=[]`，没有 final_report

**可能原因**：
1. 线上老版本代码在 report 生成失败时仍将整体状态标为 completed
2. reviewer 数据缺失导致报告跳过
3. checkpoint 命中但文件被清理

**当前代码状态**：当前版本中，report 生成失败会将 status 设为 fail（`main.py:695`），不应出现此问题。

### 1.2 线上报告缺少量化分析段 ℹ️

线上生成的报告缺少以下段（当前代码已实现）：

| 段落 | 状态 | 说明 |
|------|------|------|
| 速读卡片 | ✅ 已有 | 线上无 |
| 综合推荐等级 | ✅ 已有 | 线上无 |
| 类型标签 | ✅ 已有 | 线上无 |
| 阅读体验量化分析 | ✅ 已有 | 线上无 |
| 文本质感量化分析 | ✅ 已有 | 线上无 |
| 评分可信度报告 | ✅ 已有 | 线上无 |
| 情感曲线分析 | ✅ 新增 | 本次添加 |
| 可读性评分 | ✅ 新增 | 本次添加 |

**原因**：线上运行的 Docker 镜像是旧版本代码编译的。

**解决方案**：重新部署最新代码编译的 Docker 镜像。

### 1.3 通用报告缺少精细的量化评分 ⚠️

线上通用报告中，评分完全依赖 LLM 文字描述，没有：
- 数值化的雷达评分（0-10）
- 程序化规则评分交叉验证
- 置信度标注

当前代码已全部实现。

## 二、本次新增功能

### 2.1 sentiment_arcs.py（情感曲线引擎）

- 内置约 350 个中文情感词（正面 + 负面）
- 按章节计算情绪极性 [-1, +1]
- 自动归类为 6 种经典叙事弧线
- ASCII 情感曲线可视化
- 可信度标注

### 2.2 readability_scorer.py（可读性评分引擎）

- 综合可读性指数 0-100
- 6 维度加权：句长 + 句长方差 + 对话平衡 + 段落长度 + 连接词密度 + 信息密度
- 等级标签 + ASCII 进度条

### 2.3 集成

- `general_scan.py`：在数据采集阶段自动运行
- `report.py`：在通用报告模板中插入新段
