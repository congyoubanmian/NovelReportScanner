# 扫书优化 V3 路线图 — 感性量化 × 代码可计算的"文学感觉"

> 生成时间：2026-06-11
> 状态：规划中

---

## 核心思路

写作和阅读是"感性"的，但可以转化为**可计算的数学指标**：

| 感性体验 | 量化方法 | 数学基础 |
|---------|---------|---------|
| "文笔好不好" | 模板词密度 + 字符多样性 + 句长方差 | 统计学（频率/方差/信息熵） |
| "节奏好不好" | 章节长度均匀度 + 张力曲线斜率 + 节奏切换频率 | 时间序列分析（标准差/一阶差分） |
| "情绪被调动" | 情感词分布 + 正负情绪比率 + 情绪波动幅度 | 情感分析（词频/极性/振幅） |
| "读起来顺不顺" | 对话/叙述比 + 段落长度分布 + 连接词密度 | 信息论（香农熵）+ 统计分布 |
| "有没有深度" | 信息密度 + 伏笔回收率 + 语义层丰富度 | 信息论 + 图论 |
| "值不值得看" | 加权综合评分 + 可信度标注 + 同类对比 | 贝叶斯估计 + 置信区间 |

## V3 新增功能

### 1. 📈 情感曲线引擎 (`sentiment_arcs.py`)

**纯规则，零 LLM 成本**

从原文直接分析每章/每段的情感走向：

- **正负情绪词典**：内置约 800 个中文情感词（正面 400+ / 负面 400+）
- **章节情绪值**：正词频次 - 负词频次 / 总词频 → [-1, +1] 的情绪极性
- **情绪波动量**：相邻章节情绪差的标准差（波动大 = 情绪调动强）
- **情绪轨迹类型**：自动归类为 6 种经典叙事弧线
  - `rags_to_riches`：持续上升
  - `riches_to_rags`：持续下降
  - `man_in_hole`：先降后升
  - `icarus`：先升后降
  - `cinderella`：升→降→升
  - `oedipus`：降→升→降
- **可信度**：基于有效章节覆盖率和词典匹配率

输出：
```
【情感曲线分析】
  情绪轨迹类型：先苦后甜型 (Man in Hole)
  全书情绪极性：+0.32（偏正面）
  情绪波动量：±0.41（中等波动）
  最压抑段落：第12-15章（-0.58）
  最振奋段落：第45-48章（+0.72）
  章节覆盖率：92%（可信度高）
```

### 2. 📊 可读性评分引擎 (`readability_scorer.py`)

**纯规则，量化"好不好读"**

结合中文文本特征的可读性评估：

- **平均句长**：句子字数均值和方差
- **段落复杂度**：段落长度均值 + 长段占比
- **词汇复杂度**：单字词 vs 多字词比率、生僻字比率
- **连接词密度**：因果/转折/递进等逻辑连接词频率
- **对话平衡度**：对话占比偏离 20-30% 理想区间的程度
- **信息密度**：去重字符/总字符 × 对话信息贡献率
- **综合可读性指数**：加权公式 → 0-100 分
  - `80-100`：通俗易懂，爽文节奏
  - `60-80`：正常阅读体验
  - `40-60`：需要一定耐心
  - `0-40`：晦涩难懂或信息过载

### 3. 📋 报告综合质量评分

在速读卡片后增加**综合质量评分段**：

```
【综合质量评分】
  📊 总分：72/100（B+ 级）
  
  细分：
  ├─ 可读性：  78/100  🟢
  ├─ 情绪调动： 71/100  🟡
  ├─ 节奏把控： 68/100  🟡
  ├─ 角色塑造： 75/100  🟢（基于LLM）
  ├─ 世界观：  70/100  🟡（基于LLM）
  └─ 信息密度： 82/100  🟢
  
  置信度说明：
  🟢 = LLM与规则评分一致（高可信）
  🟡 = 仅一路评分或偏差中等
  🔴 = 数据不足或严重矛盾
```

### 4. 📐 叙事张力曲线可视化增强

在现有 ASCII 张力曲线基础上增加：

- **三幕结构标注**：自动将全书划分为"起→承→转→合"四段
- **关键转折点标记**：张力曲线中斜率突变最大的点
- **节奏标签统计**：每段的节奏标签（快/慢/高潮/过渡）分布

### 5. 🔍 同类作品量化对比

基于量化指标的横向对比：

- 与项目中其他已扫描作品在相同维度上的百分位排名
- 例如："文笔在已扫描的 22 本中排第 8 名（前 36%）"

## 设计原则

1. **不依赖纯算法解决感性问题** — 用规则统计提供基线，LLM 提供语义理解，两者交叉验证
2. **量化有边界** — 每个指标标注可信度和样本量，不装作全知
3. **渐进式** — 所有新功能默认开启但有开关，可单独关闭
4. **零额外 LLM 成本** — 新增的分析全部基于纯规则统计
